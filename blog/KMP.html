KMP

对于判断一个字符串（B="ababacb"）是否是另一个字符串（A="ababababacb"）的子串的问题，一个最容易想到的便是暴力求解（逐个字符比较）：

①从第一个字符（i=0）开始比较

②如果不是的话则将B和第二个字符比较（i=1）

如此循环判断完整个字符串，时间复杂度是O(m*n)。很明显，我们做了m次对B的重复遍历，如果我们能除去遍历B的冗余操作，便可以大大提高运行速度。

我们怎样不对B进行遍历而就能判断这次对比是否合适呢。举个例子，用两个变量i和k分别遍历A和B，某一时刻条件如下图

经过两轮的暴力求解法后，我们会到达下面这种状态：

观察蓝色的的部分，我们之所以可以省去B的遍历，是因为在B[2,3,4]这部分内容和B'[0,1,2]是重复的。也就是子串B[0,...,k-1]的前缀和后缀的最长公共子序列的长度。这个长度便是《算法导论》这本书所说的前缀函数（π={π[0],...,π[m]},m是数组B的长度）的定义。

显然π只和B自己有关，那么如何确定π的值呢？以k=3为例（当然，此时π[0]=0,π[1]=0,π[2]=1,π[3]=2已经知道了）:

下一步，B[k+1] = B[k'+1], 所以最长公共子串的长度+1即可，即π[4]=3。但是当k=5时，B[5]≠B[3], 如下图，k'的新值应该赋为π[k'-1], 如此循环直到结束。得到B的前缀函数(π(B) = [0,0,1,2,3,0,0])。

接着我们回到A的遍历中，和求B得前缀函数类似。如果A[i]=B[k], 则 k=k+1; 如果 A[i]≠B[k]且k>0，k=π[k-1]，再对比A[i]和B[k]。如果遇到B是A中一个子字符串，但还没有遍历完，保存结果后注意要从k=π[k-1]开始下一次遍历（例如A=“ababa”,B="aba",显然子串的情况出现了两次）

代码总结一下上面的算法，真的很简单：

Code 

vector < int > nextV(string b){
    int len = (int)b.size();
    vector < int > res(len,0);
    int k=0;
    for (int i=1; i < len; i++){
        while ( k>0 && b[i] != b[k] ) k = res[k-1];
        if(b[k] == b[i]) res[i] = ++k;
    }
    
    return res;
}
vector < int > KMP(string a, string b){
    int len_a = (int)a.size();
    int len_b = (int)b.size();
    vector< int > res;
    vector< int > next = nextV(b);
    int k = 0;
    for (int i=0; i < len_a; i++) {
        while(k>0 && a[i] != b[k]) k = next[k-1];
        if(b[k] == a[i]) k++;
        if(k == len_b){
            res.push_back(i-len_b+1);
            k = next[k-1];
        }
    }
    return res;
}
